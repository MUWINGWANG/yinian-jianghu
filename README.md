# 一念江湖

> 一念之差，江湖已变。

朱砂红古风武侠单页应用 RPG。玩家扮演一名偶然获悉足以颠覆正派联盟内幕的旅人，在 30 天内周旋于五大门派之间，以每日的抉择、夜访与战斗，书写属于自己的江湖结局。

---

## 游戏世界观

### 背景

乱世之中，五大门派表面上维系着武林秩序，暗地里却各怀鬼胎。一份太虚观掌门与血煞堂密谈的秘密，意外落入玩家手中。这份秘密足以改变整个江湖的格局——但也让你成为各方势力争相拉拢乃至追杀的目标。

从驿站惊变到终局之日，30 天，30 个抉择，7 种结局。

### 好感度系统

每位 NPC 拥有四个独立维度，范围 0–100：

- **信任**：愿意将后背交给你
- **亲密**：情感上的联结与依赖
- **敬畏**：因实力或身份产生的敬重
- **忌惮**：对你心存顾忌与戒备

## 游戏玩法

每天分为三个阶段：

1. **清晨** — 阅读当日场景文本，了解事件背景
2. **日间** — 从 2 个选项中做出抉择，影响秘密扩散指数与 NPC 好感度
3. **夜间** — 可选择拜访一位 NPC 深谈（每日限一次），亲密度 +3

特殊日（第 5、14、25 天）触发**回合制战斗**：

- 玩家持青云门武学与敌人对战
- 行动点制：每招消耗 1–2 点，每回合刷新
- 战败则秘密扩散 +1，不影响游戏继续

**秘密扩散指数**（0–10）是核心压力机制，部分选择会推高它，到达 8 即触发镇压结局。

## 技术架构

### 技术栈

```
Vite 5 + Vue 3.4 (Composition API) + Pinia 2 + TypeScript 5
测试：Vitest + happy-dom
```

### 目录结构

```
src/
├── types/          # 核心类型定义
│   ├── npc.ts      # NpcId, AffinityDimension, NpcProfile
│   ├── game.ts     # EndingId, Choice, Outcome, DayEvent
│   ├── faction.ts  # FactionId, FactionProfile
│   └── combat.ts   # MartialArt, Combatant, CombatState
├── data/           # 静态游戏数据
│   ├── npcs.ts     # 12 位 NPC 档案 + 初始好感度
│   ├── factions.ts # 5 大门派档案
│   ├── martial-arts.ts # 20+ 招武学（每派至少 4 招）
│   └── events.ts   # 30 天剧情事件
├── stores/         # Pinia 状态管理
│   ├── game.ts     # 天数、秘密扩散、标记、结局
│   ├── affinity.ts # 12 × 4 维好感度矩阵
│   └── combat.ts   # 回合制战斗状态
├── composables/
│   ├── useCalendar.ts  # 日历推进 + 抉择引擎 + 夜访
│   └── useEnding.ts    # 结局判定逻辑
├── components/
│   ├── DayScene.vue        # 场景描述卡片
│   ├── ChoicePanel.vue     # 抉择按钮组
│   ├── EveningFeedback.vue # 傍晚关系波动反馈
│   ├── NightVisitPanel.vue # 夜访 NPC 选择器
│   ├── CombatView.vue      # 回合制战斗 UI
│   └── AffinityPanel.vue   # 可折叠好感度侧边栏
└── views/
    ├── GameView.vue    # 主游戏编排视图
    └── EndingView.vue  # 全屏结局展示
tests/
├── types/          # 类型完整性检验
├── data/           # 静态数据完整性（12 NPC，5 门派，20+ 武学）
├── stores/         # game store、affinity store 单元测试
└── composables/    # useCalendar 完整流程测试
```

### 状态流

```
用户选择
    ↓
useCalendar.makeChoice()
    ↓
gameStore.applyOutcome()   ←→   affinityStore.adjustAffinity()
    ↓
useCalendar.advanceToNextDay()
    ↓
day === 30 → useEnding.determineEnding() → EndingView
```

### 数据设计原则

- **静态数据只读**：`src/data/` 下所有数据对象以 `as const` 或 `readonly` 标注，运行时不可变
- **好感度有界**：`adjustAffinity` 内部对 [0, 100] 做夹断，越界值安全截断
- **秘密扩散有界**：`applyOutcome` 对 [0, 10] 做夹断
- **每日夜访唯一**：`nightVisitUsed` flag 在 `advanceToNextDay` 时重置
